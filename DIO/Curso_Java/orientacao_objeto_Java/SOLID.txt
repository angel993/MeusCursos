=== SOLID ===

----- Single Responsibility Principle -----

    - A classe deve possuir uma única responsabilidade

----- Open-Closed Principle -----

    - Objetos devem estar abertos para extensão,
    mas não fechados para modificação

----- Liskov Substitution Principle -----

    - Classes derivadas devem ser substituíveis por 
    suas classes base

----- Interface Segregation Principle -----

    - Faça interfaces refinadas que são específicas
    do cliente
    - Não importar interfaces que não sejam utilizadas
    - Interfaces mais específicas

----- Dependency Inversion Principle -----

    - Depender de abstrações e não de implementações
    - Inversão de dependência não é
    Injeção de dependencia
